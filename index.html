Adaptive Reuse: Office-to-Residential Conversions

Overview

Thesis research focusing on adaptive reuse of Pittsburgh office buildings from 1965-1985, analyzing residential conversion potential with emphasis on daylighting performance.

Research Focus

  • Parametric building iterations using RhinoPython scripts
  • Comprehensive daylighting analysis via ClimateStudio API
  • Machine learning for spatial daylight autonomy prediction
  • Evaluation of 100+ building configurations
  • Focus on 1965-1985 office typologies in Pittsburgh

Methodology

  • Automated parametric model generation for design exploration
  • Batch daylighting simulation and performance optimization
  • Building systems analysis for conversion feasibility
  • Integration of computational design with building performance metrics
  • Data-driven decision making for early-stage design

Key Challenges

  • Deep floor plates limiting natural light penetration
  • Structural grid constraints on unit layouts
  • Building systems adaptation for residential requirements
  • Balancing unit count with daylighting quality

Tools & Technologies

Rhino, Grasshopper, RhinoPython, ClimateStudio API, Machine Learning, Python, sDA/ASE Analysis

Tartan Tuesday Management System

Overview

Full-stack web application built for Carnegie Mellon's Tartan Tuesday student engagement program, serving 200+ weekly participants with real-time attendance tracking, point management, and prize redemption.

Key Features

  • Real-time attendance tracking with student ID card swipe
  • Google Sheets API integration for cloud data synchronization
  • Duplicate detection preventing multiple check-ins per day
  • Point tracking system with Free Spin logic
  • Prize redemption with automatic point deduction
  • Excel import/export for roster and historical data
  • Statistics dashboard with sortable student history
  • Responsive design for desktop and mobile use

Technical Implementation

  • Frontend: React 18 with hooks-based state management
  • Backend: Google Apps Script for serverless cloud integration
  • API: RESTful endpoints for data CRUD operations
  • Data Processing: XLSX.js for Excel file handling
  • Real-time Updates: Automatic cloud synchronization on every transaction
  • Complex Logic: Duplicate prevention, cumulative point calculation, Free Spin workflow

Impact

  • Replaced manual Excel workflow saving 5+ hours weekly
  • Zero data loss with cloud-based storage
  • Handles 200+ weekly participants efficiently
  • 3,000+ historical transaction records managed

Tools & Technologies

React, JavaScript, Google Apps Script, Google Sheets API, XLSX.js, HTML/CSS, REST API, State Management

ASHRAE Design Competition: São Paulo Public Library

Overview

Comprehensive HVAC system design for a three-story public library in São Paulo, Brazil (Climate Zone 2A). Designed innovative hybrid system integrating VRF-DOAS with renewable energy.

Performance Achievements

  • 45% energy reduction (284 to 157 kWh/m²)
  • 397,800 kWh/year solar generation (250 kW PV system)
  • Operational carbon: 101 kgCO₂/m² (from 184 baseline)
  • Energy cost: $29/m² annually

System Design

  • Primary: VRF with DOAS
  • Dehumidification: Desiccant dehumidifiers
  • Ventilation: Energy Recovery Ventilators (ERV)
  • Renewable: 250 kW rooftop PV system

Tools

ClimateStudio, Energy Modeling, ASHRAE Standards 62.1, 90.1, 55, Load Calculations, PVWatts

Small Office HVAC System Selection

Overview

Comprehensive HVAC analysis for small coworking office in San Francisco. Evaluated multiple systems, recommending mini-split VRF for optimal performance.

Key Results

  • 33% energy savings with VRF
  • Site EUI: 54 kBTU/ft² (from 81 baseline)
  • Peak cooling: 79.7 kBTU/hr
  • Peak heating: 61.3 kBTU/hr
  • Fresh air: 770.52 CFM

Tools

ClimateStudio, ASHRAE 62.1 & 90.1, Load Calculations, RenewAire CORES, Energy Modeling

Attendance Matrix Tool

Overview

Web-based tool built to automate attendance tracking for Carnegie Mellon's Environments II: Active Building Systems course. Processes Excel files from 21 class sessions for 66 students, calculating attendance percentages and generating formatted reports.

Problem Solved

  • Manual attendance tracking across 21 sessions was time-consuming and error-prone
  • Data spread across multiple Excel sheets needed consolidation
  • Percentage calculations and report generation required automation
  • Needed accessible solution for future TAs without technical background

Features

  • Upload Excel file with multiple sheets (one per class session)
  • Automatic email-based student matching across sessions
  • Real-time attendance percentage calculation
  • Sortable student roster with attendance stats
  • Export processed data to formatted Excel file
  • Client-side processing (no server required, data privacy)

Technical Implementation

  • Frontend: Pure JavaScript with HTML/CSS
  • Excel Processing: SheetJS (XLSX.js) library
  • Data Processing: Array manipulation and aggregation algorithms
  • Export: Dynamic Excel generation from processed data
  • Deployment: GitHub Pages for zero-cost hosting

Impact

  • Reduced attendance processing from hours to seconds
  • Eliminated manual calculation errors
  • Saved 5+ hours of TA administrative work
  • Now available as open-source tool for other educators
  • Live at: ananyashrimali03.github.io/attendance_matrix

Tools & Technologies

JavaScript, HTML/CSS, SheetJS (XLSX.js), Excel Processing, GitHub Pages, Automation
